# 目標
建構一個篩選美股選擇權 SELL PUT 的標的選擇系統

# 需要調研的方向
1. 標的選擇的指標
	1. 流動性指標
	2. 價內價外指標
	3. 股票現有價格
2. 其他我漏掉沒有寫的可能指標
3. 目前網路上有那些免費的 API 可以獲得相關信息

---

# 系統核心假設
SELL PUT 的標的選擇不是單純找「權利金高」的合約，而是要同時回答三個問題：

1. **我願不願意用履約價接這檔股票？**
2. **這個 Put 的報酬是否足以補償被指派風險？**
3. **如果市場短期下跌，我是否還有流動性與資金承受能力？**

因此系統應分成兩層：

1. **股票層篩選**：先決定哪些股票值得被賣 Put。
2. **選擇權合約層篩選**：再決定哪個到期日、履約價、Delta、權利金條件最適合。

# 一、股票層篩選指標

## 1. 基本流動性
避免賣在冷門股票上，否則價差大、難平倉、被動成交成本高。

可用指標：

- **股票日均成交量**：例如 30 日平均成交量 > 1,000,000 股。
- **股票日均成交金額**：股價 × 成交量，例如 > 5,000 萬美元。
- **市值**：例如 > 50 億美元，避免小型股跳空風險過高。
- **Bid-Ask Spread**：股票本身價差越小越好。

初步建議：

```text
股票 30 日均量 > 100 萬股
市值 > 50 億美元
股價 > 10 或 20 美元
```

## 2. 價格與可承接能力
SELL PUT 本質上可能會買進股票，所以履約價 × 100 必須在可承受範圍內。

可用指標：

- **現股價格**：避免價格過高導致單一合約占用資金太大。
- **履約價 × 100 / 總資金比例**：單一合約最大資金占用不宜過高。
- **是否願意長期持有**：若被指派，是否願意持有該公司或 ETF。

建議規則：

```text
單一合約最大指派金額 <= 總資金 5%～10%
若為高信心 ETF 可放寬；若為單一股票應更保守
```

## 3. 基本面品質
因為 SELL PUT 的尾端風險是「接到一路下跌的股票」，所以基本面比短線方向更重要。

可用指標：

- **營收成長率**：YoY 或 TTM 成長是否穩定。
- **EPS / 自由現金流**：是否長期為正。
- **負債比率**：高槓桿公司在景氣反轉時風險較大。
- **毛利率 / 營業利益率**：是否有穩定獲利能力。
- **ROE / ROIC**：資本效率。
- **估值**：P/E、Forward P/E、P/S、EV/EBITDA 是否過熱。

可以先用簡化版：

```text
營收 YoY 不明顯衰退
自由現金流為正
負債沒有異常上升
不是即將破產、重組、被調查的公司
```

## 4. 波動與風險事件
高權利金通常來自高波動，但高波動也代表更高被指派與跳空風險。

可用指標：

- **歷史波動率 Historical Volatility, HV**
- **隱含波動率 Implied Volatility, IV**
- **IV Rank / IV Percentile**：判斷目前 IV 是否相對歷史偏高。
- **Beta**：相對大盤的敏感度。
- **最大回撤 Max Drawdown**
- **財報日期 Earnings Date**：財報前賣 Put 權利金高，但跳空風險也高。
- **除息日 / 重大法說 / FDA / 併購 / 訴訟事件**

建議規則：

```text
若不做事件交易，避開財報前 7～14 天到期的合約
避免單日可能跳空 10% 以上且無法承受的標的
```

## 5. 技術面與價格位置
SELL PUT 通常希望履約價落在「較有支撐」的位置。

可用指標：

- **距離 52 週高低點的位置**
- **均線位置**：20D / 50D / 200D MA。
- **支撐區**：前低、成交密集區、長期均線。
- **RSI**：避免在極端弱勢中接刀。
- **趨勢狀態**：上升、盤整、下降。

簡化規則：

```text
優先選擇長期趨勢未破壞、價格仍在 200D MA 上方或接近重要支撐的標的
避免明確下降趨勢中只因 IV 高就賣 Put
```

# 二、選擇權合約層篩選指標

## 1. 選擇權流動性
這是 SELL PUT 非常重要的硬條件。

可用指標：

- **Open Interest, OI**：未平倉量。
- **Volume**：當日成交量。
- **Bid-Ask Spread**：價差越小越好。
- **Mid Price**：用 bid/ask 中間價評估合理成交。

建議規則：

```text
OI > 500 或 1,000
Volume > 100
Bid-Ask Spread / Mid Price < 5%～10%
避免只有報價但幾乎無成交的合約
```

## 2. 價內價外與 Delta
SELL PUT 通常會賣價外 Put，但價外程度要依策略而定。

常用指標：

- **Moneyness**：履約價 / 現價。
- **OTM 百分比**：`(現價 - 履約價) / 現價`。
- **Delta**：近似被指派或到期價內機率的參考，不是精確機率。
- **Probability ITM / OTM**：若 API 有提供可用。

常見區間：

```text
保守型：Delta 約 -0.10 ～ -0.20
平衡型：Delta 約 -0.20 ～ -0.30
積極型：Delta 約 -0.30 ～ -0.40
```

## 3. 到期天數 DTE
DTE 影響 Theta 收益、Gamma 風險與管理頻率。

常見選擇：

- **7～14 DTE**：Theta 快，但 Gamma 與短線跳空風險高，管理頻繁。
- **30～45 DTE**：常見 wheel / short premium 區間，時間價值與風險較平衡。
- **60+ DTE**：權利金較高但資金占用久，年化報酬未必更好。

建議起點：

```text
優先研究 30～45 DTE
避開財報或重大事件落在持倉期間的合約
```

## 4. 權利金與報酬率
不要只看權利金絕對值，要看相對風險與資金占用。

可用公式：

```text
現金擔保報酬率 = 權利金 / 履約價
年化報酬率 = 權利金 / 履約價 × 365 / DTE
安全邊際 = (現價 - 損益兩平價) / 現價
損益兩平價 = 履約價 - 權利金
```

應同時比較：

- 年化報酬是否足夠。
- 損益兩平價是否落在可接受位置。
- 權利金是否只是來自即將財報或重大風險。

## 5. IV 與波動溢價
SELL PUT 是賣波動，理想情況是「IV 偏高，但標的品質可接受」。

可用指標：

- **IV Rank / IV Percentile**
- **IV - HV 差距**
- **同標的不同履約價 IV skew**
- **Put IV 是否明顯高於 Call IV**：代表市場下跌保護需求高。

建議：

```text
優先找 IV Rank 偏高，但不是因單一災難事件導致的標的
若 IV 高但基本面惡化，應降低排序或排除
```

# 三、可以補充的風險指標

## 1. 集中度風險
避免所有 Put 都集中在同一產業或同一因子。

可用分類：

- 產業：科技、金融、能源、醫療、消費等。
- 資產類型：個股、ETF、指數 ETF。
- 因子：高 Beta、AI 概念股、小型股、利率敏感股。

規則範例：

```text
單一標的最大風險 <= 總資金 5%～10%
單一產業最大風險 <= 總資金 20%～30%
```

### 行業分類資料

美股多數「上市公司普通股」都能取得行業分類，但不是所有 ticker 都一定有乾淨分類。

常見分類標準：

- **GICS**：常見於投資分析，分成 sector、industry group、industry、sub-industry。
- **ICB**：另一套金融市場分類。
- **SIC**：美國政府與 SEC filing 常見的產業代碼。
- **NAICS**：北美產業分類，偏政府統計用途。

需要注意：

- 一般大型美股公司通常會有 sector / industry。
- ETF、ETN、基金、SPAC、ADR、特殊目的公司，分類可能缺失或不適合直接與普通股混用。
- 不同資料源的 sector / industry 名稱可能不同，系統內應選一個主資料源並正規化。

可取得來源：

| 來源 | 可取得內容 | 適合程度 |
|---|---|---|
| yfinance | sector、industry；來源是 Yahoo Finance，分類名稱通常接近 Morningstar sector / industry，而不是官方 GICS code | 免費 prototype，但穩定性普通 |
| Finnhub | company profile，含 sector / industry，部分資料含 SIC / NAICS | 適合 MVP |
| Financial Modeling Prep | sector、industry、SIC classification，可透過線上 API 取得 available sectors、available industries、company profile、stock screener、SIC industry classification | 適合基本面與分類資料 |
| Alpha Vantage | OVERVIEW endpoint 含 Sector、Industry | 可作備援，但免費額度有限 |
| SEC company facts / filings | SIC code | 權威但需要自己整理 |
| Nasdaq / exchange symbol directory | 上市資訊，通常不一定有完整行業分類 | 適合建立股票池，不適合單獨做分類 |

yfinance / Yahoo Finance 分類特性：

```text
資料來源：Yahoo Finance
常見分類體系：接近 Morningstar 的 sector / industry 分類
不是：官方 GICS code、SIC code、NAICS code
```

常見 sector 名稱可能包含：


使用方式範例：

```python

```

注意：

```text

3. 適合 MVP 或備援，不建議當作唯一權威分類來源。
4. 若要做產業曝險控管，要先把 yfinance 的分類名稱映射到系統內部標準。
```

對 Sell Put 系統的用途：

```text
1. 控制單一產業曝險
2. 避免候選清單全部集中在高 Beta 科技股
3. 對同行業標的做相對比較，例如只選同產業中流動性與基本面較好的公司
4. 設計產業風險警示，例如金融股、半導體、能源股同時承壓
```

## 2. 大盤環境
SELL PUT 在大盤急跌時會同時承壓，因此需要市場 regime filter。

可用指標：

- **SPY / QQQ 是否在 200D MA 上方**
- **VIX 水準與趨勢**
- **市場寬度**：例如 S&P 500 成分股在 50D MA 上方比例。
- **利率與美元趨勢**：影響成長股與風險資產估值。

簡化規則：

```text
若 VIX 急升且 SPY 跌破 200D MA，降低賣 Put 數量或提高現金比例
```

## 3. 下跌情境壓力測試
對每個候選合約估算如果股價下跌會發生什麼事。

情境：

```text
標的下跌 5%、10%、20%
大盤下跌 5%、10%
IV 再上升 20%～50%
```

要回答：

- 是否會被指派？
- 指派後部位占總資金多少？
- 是否有足夠現金接股？
- 若不接股，是否能承受買回損失？

## 4. 稅務與交易成本
雖然不是標的指標，但會影響實際報酬。

要記錄：

- 手續費與合約費。
- Bid-Ask 成本。
- 短期資本利得稅。
- 匯率與資金成本。

# 四、候選標的排序模型

可以先用打分制，不必一開始就做複雜模型。

## 股票層分數

| 類別 | 指標 | 權重 |
|---|---|---:|
| 流動性 | 股票成交量、市值、價差 | 20% |
| 品質 | 營收、FCF、負債、ROIC | 25% |
| 估值 | P/E、P/S、相對歷史估值 | 15% |
| 風險 | Beta、HV、財報、事件風險 | 20% |
| 技術面 | 趨勢、支撐、均線位置 | 20% |

## 合約層分數

| 類別 | 指標 | 權重 |
|---|---|---:|
| 選擇權流動性 | OI、Volume、Bid-Ask Spread | 25% |
| 安全邊際 | OTM%、Delta、損益兩平價 | 25% |
| 報酬 | 權利金、年化報酬 | 20% |
| 波動溢價 | IV Rank、IV-HV | 20% |
| 事件風險 | 財報、重大事件 | 10% |

## 最小可行篩選流程 MVP

```text
1. 從 SPY / QQQ / liquid large caps 建立股票池
2. 排除低流動性、過小市值、財報風險標的
3. 抓 30～45 DTE Put chain
4. 篩 Delta -0.15 ～ -0.30 的 OTM Put
5. 篩 OI、Volume、Bid-Ask Spread
6. 計算 cash-secured return、annualized return、breakeven
7. 依基本面品質 + 合約報酬 + 安全邊際排序
8. 輸出 Top N 候選清單與風險說明
```

# 五、免費或低成本 API / 資料來源

> 注意：選擇權即時資料通常有交易所授權成本，真正「免費、穩定、即時、完整 Greeks」很少見。免費方案適合研究、初篩與個人小工具；正式下單前仍應以券商報價確認。

## 實測資料源：usataco.top 美股期權策略系統

> 實測時間：2026-06-02。此網站是前端 React SPA，頁面資料主要透過 `/api/*` 取得。API 穩定性普通，測試中偶爾出現 `504 Gateway Time-out`，需要重試與錯誤處理。

用途：

- 猛猛機會：高漲幅、高 IV 個股與對應 Put 合約初篩
- 穩穩機會：較穩健的 Sell Put 候選清單
- 開倉評估：輸入股票代號後做正股六維評分與 Sell Put 合約評估
- 分析歷史：取得最近分析過的股票代號

### 認證 / Header

前端登入後會把 username 存到 localStorage：

```text
localStorage key: usataco.username
```

後續 API 主要靠 HTTP header 傳使用者名稱：

```bash
-H 'Content-Type: application/json'
-H 'X-Username: <username>'
```

登入 API：

```bash
POST https://usataco.top/api/login
Content-Type: application/json

{
  "username": "<username>",
  "password": "<password>"
}
```

成功回傳範例：

```json
{
  "message": "欢迎，<username>！",
  "success": true,
  "username": "<username>"
}
```

> 安全注意：筆記中不要保存明文密碼；程式應從環境變數或安全密鑰管理讀取。

### API 1：猛猛機會

頁面：

```text

```


Market Data Production 中與期權有關的端點：

| 功能 | Endpoint | 對 Sell Put 系統的用途 |
|---|---|---|
| Option Chain | `GET /marketdata/v1/chains` | 取得某標的完整或篩選後的選擇權鏈，是 Sell Put 篩選器最核心的 endpoint |
| Option Expiration Chain | `GET /marketdata/v1/expirationchain` | 取得可用到期日，先找 30～45 DTE，再去抓 chain |
| Quotes | `GET /marketdata/v1/quotes` | 查股票或特定 option contract 的即時 quote，例如 bid / ask / last / mark |
| Price History | `GET /marketdata/v1/pricehistory` | 取得標的歷史價格，用來算 HV、均線、支撐、趨勢 |
| Movers | `GET /marketdata/v1/movers/{symbol_id}` | 輔助觀察市場異動，不是 Sell Put 必要 endpoint |
| Market Hours | `GET /marketdata/v1/markets` 或相關 market hours endpoint | 確認市場是否開盤，避免非交易時間誤判 quote |
| Instruments | `GET /marketdata/v1/instruments` | 查 symbol / CUSIP / instrument 資訊，輔助建立股票池 |

`/marketdata/v1/chains` 常用參數：

```text
symbol：標的，例如 AAPL、SPY
contractType：CALL / PUT / ALL；Sell Put 主要用 PUT
strikeCount：回傳 ATM 上下多少個履約價
includeUnderlyingQuote：是否包含標的股票 quote
strategy：SINGLE / ANALYTICAL / COVERED / VERTICAL / STRADDLE 等；Sell Put 初期用 SINGLE，模擬分析可用 ANALYTICAL
strikeRange：ITM / OTM / ALL；Sell Put 初篩可用 OTM 或 ALL
fromDate / toDate：到期日區間，用來抓 30～45 DTE
expMonth：指定到期月份或 ALL
optionType：standard / non-standard / ALL，依文件實際值為準
```

`strategy=ANALYTICAL` 時可用的模擬參數：

```text
volatility：自訂波動率，測 IV 變化
underlyingPrice：自訂標的價格，測股價變動
interestRate：自訂利率
daysToExpiration：自訂到期天數，測 Theta / 時間衰減
```

建議抓取流程：

```text
1. 用 expirationchain 取得指定 symbol 的所有到期日
2. 篩出 30～45 DTE
3. 用 chains 抓 PUT option chain
4. 篩 Delta -0.15～-0.30、OTM、OI、Volume、Spread%
5. 用 quotes 補最新 bid / ask / mark
```

### API 路線的優勢與劣勢

優勢：

- 適合建立穩定、可重現的 SELL PUT 選擇系統。
- 可以明確計算所有監控指標，例如 breakeven、DTE、Delta、OI、Spread%、cash required。

劣勢：

- 初期開發成本較高。
- 需要處理 OAuth / token / rate limit / error handling。
- 每個功能都要自己設計資料結構與流程。

### MCP 路線的優勢與劣勢

優勢：

- 適合快速讓 AI 查詢 Tradier 資料，例如：「找出 AAPL 30～45 DTE、Delta 約 -0.2 的 Put」。
- 適合互動式研究、解釋選擇權鏈、比較候選合約。
- 開發門檻可能比直接串 API 低。

劣勢：

- 不適合一開始就拿來做完全自動交易。
- AI 可能誤解自然語言指令，因此需要人類確認。
- 監控指標、交易日誌、回測資料仍應落地到自己的資料庫或筆記系統。
- 權限控管更重要，尤其是涉及下單、改單、取消單。

### 對本系統的建議

```text
正式系統核心：優先用 API
AI 輔助研究：可以用 MCP
下單：初期一律人工確認，不讓 MCP 直接自動下單
```

建議分工：

```text
API：每日批次抓 options chain、quote、account、position，產生候選清單與風控指標
MCP：讓 AI 幫忙解釋候選清單、比較合約、產生交易前檢查問題
人工：最後確認是否進場、是否展期、是否接股
```

官方文件：

- Tradier MCP：https://docs.tradier.com/docs/tradier-mcp

### Tradier vs Charles Schwab API

| 面向 | Tradier | Charles Schwab |
|---|---|---|
| 定位 | API-first / options trader 友善的券商與資料服務 | 大型傳統券商，API 是帳戶與交易服務的延伸 |
| 適合對象 | 想做程式交易、AI agent、options 工具整合的人 | 已經用 Schwab / thinkorswim 管理資產的人 |
| 選擇權資料 | Options chain、quote、Greeks、IV 設計較直接 | Options chain、quote、帳戶資料完整，但開發體驗可能較券商導向 |
| 帳戶整合 | 需要 Tradier 帳戶或 token 權限 | 使用既有 Schwab 帳戶、持倉、buying power |
| 開發體驗 | 文件與 endpoint 通常較簡潔，適合快速串接 | OAuth 與帳戶授權流程較正式，可能較繁瑣 |
| AI 整合 | 有 Tradier MCP，適合 AI assistant 直接查詢與輔助分析 | 主要走 API；若要 MCP 通常需自己包一層 |
| 台灣使用者風險 | 需確認是否能開戶、取得 market data 與交易權限 | 你已有 Schwab 帳戶是優勢，但 Developer Portal / 2FA 可能卡台灣門號 |
| 交易成本 | 需查 Tradier 當前股票 / options commission 與方案 | Schwab 美股通常免股票佣金，options 常見為每合約費用，需以帳戶實際費率為準 |
| 下單自動化 | 較適合程式化交易與第三方整合 | 較適合連接既有主帳戶，但自動化前需處理 OAuth 與風控 |
| 對本系統角色 | 候選資料源 + AI 輔助研究 + 可能的交易執行環境 | 若可成功申請 API，適合作為主帳戶監控與下單前檢查資料源 |

#### 選擇建議

```text
如果目標是最快做出 Sell Put 篩選器：
優先測 Tradier API / MCP，因為 options endpoint 與 AI 整合較直接。

如果目標是監控你真正的帳戶與資金：
優先測 Charles Schwab API，因為它能接近你的實際 cash、buying power、positions。

如果 Schwab Developer Portal 卡台灣手機驗證：
Schwab 降級為人工確認與券商操作介面；資料與研究先用 Tradier / yfinance / marketdata.app。
```

#### 對 Sell Put 系統的實務分工

```text
Tradier：
- 抓 options chain
- 快速查 Greeks / IV / OI / volume
- 讓 AI 用 MCP 輔助比較合約

Charles Schwab：
- 查實際帳戶 cash / buying power
- 查持倉與是否被指派
- 下單前用實際券商資料做最後確認
```

結論：

```text
Tradier 比較像「選擇權資料與自動化開發工具」。
Charles Schwab 比較像「你的實際券商帳戶與資金控制中心」。

本系統不必二選一：
用 Tradier 做研究與候選篩選，用 Schwab 做帳戶確認與最終交易，是比較穩健的架構。
```

# 七、從 Sell Put 核心理解轉成監控指標

這一段把測驗中的 10 個核心觀念，以及目前答錯題目暴露出的盲點，轉成系統需要監控的欄位與警示規則。

## 1. 目前理解盲點對應的監控指標

| 盲點 | 正確觀念 | 系統要監控的指標 | 用途 |
|---|---|---|---|
| 把履約價誤認為實際成本 | 被指派成本 = 履約價 - 權利金 | `breakeven`、`premium_received`、`assignment_cost` | 判斷真正接股成本與安全邊際 |
| 沒有把 Theta 當成賣方優勢 | 時間衰減通常有利於 Sell Put 賣方 | `theta`、`theta_per_day`、`theta_yield`、`days_to_expiration` | 評估時間價值流失是否足以補償風險 |
| 股價跌破履約價時只想到等到期 | 可以買回、展期、接股或調整 | `distance_to_strike`、`itm_status`、`roll_candidate_score`、`unrealized_pnl` | 提前產生部位管理訊號 |

## 2. 十個 Sell Put 核心問題轉成指標

| 題目核心 | 策略重點 | 需要監控的指標 | 建議規則 |
|---|---|---|---|
| 1. Sell Put 承擔買股義務 | 賣方收權利金，但要準備以履約價買 100 股 | `cash_required = strike × 100`、`cash_coverage_ratio` | cash-secured put 應保留足額現金 |
| 2. 對標的股票的看法 | 適合偏多或中性，不適合看空標的 | `trend_score`、`support_score`、`fundamental_score` | 只賣願意接股的標的 |
| 3. 盈虧結構 | 最大獲利有限，最大損失接近履約價 × 100 | `max_profit`、`max_loss_estimate`、`reward_risk_ratio` | 不因權利金高而忽略尾端風險 |
| 4. 實際持股成本 | 權利金會降低損益兩平點 | `breakeven = strike - premium`、`breakeven_margin_pct` | breakeven 應低於合理支撐或可接受買入價 |
| 5. CSP 現金擔保 | 新手優先使用 Cash-Secured Put | `reserved_cash`、`cash_utilization`、`portfolio_buying_power` | 單一合約指派金額不超過總資金 5%～10% |
| 6. Theta 衰減 | 時間是賣方朋友，但不是無風險收益 | `theta`、`theta_to_delta_risk_ratio`、`extrinsic_value` | Theta 收益不足以抵消下跌風險時降低排序 |
| 7. IV 高低 | IV 高代表權利金高，也代表風險高 | `iv`、`iv_rank`、`iv_percentile`、`iv_hv_spread` | 優先找 IV 偏高但非災難事件的標的 |
| 8. 到期價內會被指派 | 收盤價低於履約價通常有指派風險 | `probability_itm`、`delta_abs`、`distance_to_strike_pct` | 越接近履約價越需要管理 |
| 9. 跌破履約價的處理 | 不想接股時要評估展期或停損 | `roll_yield`、`roll_down_available`、`debit_to_close`、`new_credit` | 到期前若 ITM 且不想接股，產生 roll / close alert |
| 10. Wheel 策略閉環 | Sell Put 接股後可轉 Sell Covered Call | `assigned_shares`、`covered_call_candidate`、`cost_basis` | 被指派後自動追蹤 covered call 條件 |

## 3. 建議新增的核心計算欄位

```text
premium = mid 或實際成交權利金
cash_required = strike × 100
breakeven = strike - premium
assignment_cost = breakeven × 100
otm_pct = (underlying_price - strike) / underlying_price
breakeven_margin_pct = (underlying_price - breakeven) / underlying_price
cash_secured_return = premium / strike
annualized_return = premium / strike × 365 / DTE
spread_pct = (ask - bid) / mid
intrinsic_value = max(strike - underlying_price, 0)
extrinsic_value = option_price - intrinsic_value
unrealized_pnl = entry_credit - current_option_mid
close_cost_ratio = current_option_mid / entry_credit
```

## 4. 盲點修正用警示設計

### A. 成本認知警示

目的：避免只看履約價，忘記權利金已降低成本。

```text
顯示：
- 履約價
- 收到權利金
- 實際損益兩平點
- 若被指派的總成本
```

範例：

```text
Strike = 175
Premium = 3
Breakeven = 172
Assignment Cost = 17,200 美元
```

### B. Theta 收益警示

目的：把「時間衰減對賣方有利」轉成可觀察收益。

```text
theta_yield = abs(theta) / option_mid
theta_to_delta_risk_ratio = abs(theta) / abs(delta)
```

解讀：

- Theta 越高，時間衰減越快。
- 但若 Delta 也很高，代表股價方向風險更大。
- 不應只因 Theta 高就進場。

### C. 跌破履約價管理警示

目的：避免等到到期才發現只能被動接股。

觸發條件：

```text
underlying_price <= strike × 1.03
或 probability_itm > 40%
或 delta_abs > 0.40
或 unrealized_pnl < -entry_credit × 1.5
```

產生動作建議：

```text
1. 若願意接股：維持，但確認現金足夠。
2. 若不願意接股：評估買回停損。
3. 若仍看好但想降低壓力：評估 roll out / roll down。
```

## 5. 進場前檢查清單

每一筆 Sell Put 進場前都要讓系統回答：

```text
1. 我願意用 breakeven 價格買這檔股票嗎？
2. 如果今天就被指派，我有足夠現金嗎？
3. 單一標的最大指派成本占總資金多少？
4. 權利金帶來的年化報酬是否足以補償下跌風險？
5. OI、Volume、Bid-Ask Spread 是否足夠好？
6. DTE 是否落在策略範圍，例如 30～45 天？
7. Delta 是否落在策略範圍，例如 -0.15～-0.30？
8. IV 高是正常波動溢價，還是財報 / 訴訟 / 災難事件造成？
9. 若股價跌破履約價，我要接股、停損還是展期？
10. 若被指派，是否能進入 covered call / wheel 下一步？
```

## 7. 簡化版決策規則

```text
若合約尚未進場：
- 只選願意用 breakeven 接股的標的
- 只選有足額現金擔保的合約
- 優先選 Delta -0.15 ～ -0.30、DTE 30～45、OI 足夠、Spread 小的 Put

若合約已進場：
- 獲利達原始權利金 50%～80% 時，考慮買回平倉
- 價格接近履約價時，提前評估是否願意接股
- 若不願接股，不要等到最後一天才處理
- 若選擇展期，要確認新合約不是單純把虧損延後
```
